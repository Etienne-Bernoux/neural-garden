## Règle de workflow — ordre obligatoire

Le workflow suit un ordre strict. **Chaque phase doit être complétée avant de passer à la suivante.**

```
Brainstorm → Plan → Work → Review → Compound
```

### Gardes

- **INTERDIT** de sauter une phase. Pas de Plan sans Brainstorm terminé. Pas de Work sans Plan validé par Etienne.
- **INTERDIT** de fusionner des phases (ex: proposer un plan pendant le brainstorm).
- Chaque phase a un **livrable explicite** qu'Etienne doit valider avant de passer à la suite :
  - Brainstorm → décisions claires, périmètre défini
  - Plan → tâches découpées avec critères de done, validées par Etienne
  - Work → code implémenté et testé par les sous-agents
  - Review → findings présentés, corrections tranchées par Etienne
  - Compound → doc à jour, commit propre

### En cas de doute

Si Claude n'est pas sûr de la phase courante, il demande : "On est en phase [X], je continue ou tu veux revenir en arrière ?"
